# Mavzu 19: Dynamic Programming (kirish) 🧩

DP = katta masalani kichik qism-masalalarga bo'lib, javoblarini **saqlab qayta ishlatish** (shu bilan rekursiyaning takror hisoblashini yo'qotamiz). Odatda `dp[]` massiv to'ldiriladi.

> Difficulty: 🟢 oson · 🟡 o'rta · 🔴 qiyin

---

## ✅ Yechilgan misol: Zinapoyaga chiqish yo'llari soni

> **Masala:** `n` pog'onali zinapoya. Har qadamda **1 yoki 2** pog'ona ko'tarilasiz. Necha xil yo'l bilan tepaga chiqish mumkin?
> `n = 5` → `8`

### 1-qadam: O'QI va TUSHUN
- **Input:** son `n` (pog'onalar soni)
- **Output:** yo'llar soni
- **Kalit:** oxirgi qadam yo `1` pog'ona, yo `2` pog'ona edi — demak javob shu ikki holatning yig'indisi.

### 2-qadam: QO'LDA YECH
Kichik holatlardan boshlayman:
- `n=1` → faqat `1` yo'l → **1**
- `n=2` → `1+1` yoki `2` → **2**
- `n=3` → `dp[2] + dp[1] = 2 + 1` = **3**
- `n=4` → `dp[3] + dp[2] = 3 + 2` = **5**
- `n=5` → `dp[4] + dp[3] = 5 + 3` = **8** ✅

Aniq Fibonachchiga o'xshaydi!

### 3-qadam: REJA
1. `dp` massiv: `dp[1]=1`, `dp[2]=2`
2. `i` ni `3` dan `n` gacha yuritaman:
   - `dp[i] = dp[i-1] + dp[i-2]`
3. `dp[n]` qaytaraman

### 4-qadam: KOD
```javascript
function climbStairs(n) {
  if (n <= 2) return n;
  const dp = [];
  dp[1] = 1;
  dp[2] = 2;
  for (let i = 3; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];   // oldingi 2 holatning yig'indisi
  }
  return dp[n];
}
```

### 5-qadam: TEKSHIR
- `climbStairs(5)` → `8` ✅
- `climbStairs(2)` → `2` ✅
- `climbStairs(1)` → `1` ✅

> 🔑 **PATTERN — "dp[] to'ldir":** javobni kichik holatlardan boshlab massivga yozing, har keyingisini oldingilaridan hisoblang. Asosiy savol — "`dp[i]` ni `dp[i-1]`, `dp[i-2]`... orqali qanday yozaman?". Boshlang'ich (base) qiymatlarni to'g'ri bering.
